# 微信小程序 - AI 图片生成 需求分析文档

## 一、项目概述

本项目为基于微信小程序的 AI 图片生成应用，用户通过输入文字描述生成对应图片，图片生成能力由 Coze API 提供。

---

## 二、功能需求

### 2.1 核心功能

**文字转图片**

- 用户可输入文字描述，作为生成图片的提示词
- 系统根据用户输入调用 Coze API 完成图片生成
- 生成结果以图片形式展示给用户

**生成结果操作**

- 图片生成完成后，用户可选择以下操作：
  - **保存到本地相册**：将当前生成的图片保存至用户设备相册
  - **重新生成**：基于当前提示词与类型，重新调用 API 生成新图片

### 2.2 图片类型选择

- 支持用户选择生成的图片风格/类型
- 可选类型共三种：
  - **人物**：侧重人物、肖像类画面
  - **动漫**：侧重动漫、二次元风格
  - **通用**：不限定风格，适配多种场景
- 默认选中类型为「通用」

### 2.3 提示词案例

- 提供若干预设提示词案例供用户使用
- 用户可选择其中任意一条作为当前输入
- 案例需覆盖不同风格，便于快速体验与参考

### 2.4 历史记录

- 不保留用户生成的图片历史记录
- 每次使用以「当次生成」为主，无需历史列表、收藏等功能

---

## 三、用户角色

- **普通用户**：使用小程序输入文字、选择类型和案例，并获取生成图片

---

## 四、使用流程（业务层面）

1. 用户打开小程序
2. 用户输入或选择提示词
3. 用户选择图片类型（人物 / 动漫 / 通用，默认通用）
4. 用户发起生成
5. 系统调用 Coze API 生成图片
6. 用户查看生成结果
7. 用户可选择：保存至本地相册，或重新生成

---

## 五、非功能需求（业务约束）

### 5.1 接口依赖

- 图片生成依赖 Coze API
- 需具备与 Coze 服务对接的能力

### 5.2 运行环境

- 基于微信小程序平台
- 遵循微信小程序相关规范与限制

---

## 六、需求边界说明

- 本文档仅描述业务与产品需求
- 不涉及具体技术选型、架构设计及实现方案
