# 微信小程序 - AI 图片生成 技术方案

## 一、文档说明

本文档基于《需求分析文档》中的功能需求与非功能需求，描述项目的技术选型、架构设计与实现方案。不包含具体代码实现，仅阐述技术层面的设计方案。

---

## 二、技术选型

### 2.1 开发框架

- **微信原生小程序**：采用微信官方小程序原生框架进行开发
- **原因**：轻量、无第三方依赖、与微信生态集成好，符合当前功能规模

### 2.2 请求与接口

- 使用微信小程序内置 `wx.request` 进行网络请求
- 通过配置合法域名实现与 Coze API 的跨域通信

### 2.3 图片展示

- 使用 `image` 组件展示生成的图片
- 支持加载中、失败等状态的占位与反馈

---

## 三、整体架构

### 3.1 分层架构

```
┌─────────────────────────────────┐
│         页面层 (Pages)            │  ← 用户交互、UI 展示
├─────────────────────────────────┤
│         逻辑层 (Logic)            │  ← 业务编排、状态管理
├─────────────────────────────────┤
│         服务层 (Service)          │  ← API 调用、数据封装
├─────────────────────────────────┤
│         配置层 (Config)           │  ← 常量、提示词、类型定义
└─────────────────────────────────┘
```

### 3.2 数据流

1. 用户输入/选择 → 页面收集
2. 页面调用逻辑层 → 校验、组装请求参数
3. 逻辑层调用服务层 → 请求 Coze API
4. 服务层返回结果 → 逻辑层处理 → 页面展示

---

## 四、Coze API 对接方案

### 4.1 对接方式

- 在小程序端直接请求 Coze 图片生成相关 API 端点
- 需在 Coze 平台配置 API Token 或密钥，通过安全方式传递（如环境变量、云函数代理等，视部署方式而定）

### 4.2 请求参数设计

- **提示词 (prompt)**：用户输入的文本或所选案例内容
- **图片类型 (style)**：`character` / `anime` / `general` 对应人物、动漫、通用
- 根据 Coze 实际 API 文档补充其他必填/可选参数

### 4.3 响应与异常

- 成功：返回图片 URL 或 base64 数据，供 `image` 组件展示
- 失败：统一错误提示，不落库、无历史记录

## 五、功能模块设计

### 5.1 页面模块

| 模块         | 职责                             |
|--------------|----------------------------------|
| 首页/生成页  | 输入框、类型选择、案例选择、生成按钮、结果展示、保存至相册、重新生成 |

单页应用即可满足需求，无需多页面路由。

### 5.2 逻辑模块

| 模块       | 职责                                             |
|------------|--------------------------------------------------|
| 生成逻辑   | 校验提示词、组装请求参数、调用服务、处理 loading/结果 |
| 类型管理   | 维护人物/动漫/通用选项及默认选中状态             |
| 案例管理   | 维护预设提示词列表，支持点击填充到输入框         |
| 结果操作   | 保存至本地相册、重新生成（复用当前参数再次调用） |

### 5.3 服务模块

| 模块        | 职责                                   |
|-------------|----------------------------------------|
| Coze 服务   | 封装 Coze 图片生成 API 的请求与响应解析 |

### 5.4 配置模块

| 配置项     | 说明                                         |
|------------|----------------------------------------------|
| API 地址   | Coze 图片生成接口 URL                         |
| 图片类型   | 人物、动漫、通用及其映射关系                  |
| 提示词案例 | 若干预设文案及所属类型（可选）                |

## 七、关键技术点

### 7.1 图片类型与 Coze 参数映射

- 前端维护类型选项：人物、动漫、通用
- 请求时转换为 Coze 可识别的参数值，保证类型与生成效果一致

### 7.2 提示词案例数据结构

- 建议结构：`{ id, text, type? }`，`type` 可选用于与图片类型联动
- 存储于 `config/prompts.js`，便于维护与扩展

### 7.3 加载与异常处理

- 生成过程中展示 loading，防止重复点击
- 网络异常、API 错误统一提示，不持久化
- 无历史记录，无需本地存储

### 7.4 结果操作实现

- **保存至本地相册**：使用 `wx.saveImageToPhotosAlbum` 将生成的图片保存至用户相册，需用户授权相册写入权限
- **重新生成**：保留当前提示词与图片类型，再次调用 Coze API 生成新图，流程与首次生成一致，展示 loading 后替换结果区域

### 7.5 微信小程序限制

- 需在微信公众平台配置 Coze API 的 request 合法域名
- 关注图片 URL 的合法性与缓存策略
- 若使用 base64，需考虑体积与性能

## 八、安全与合规

- API 密钥不硬编码在前端，通过服务端或云函数转发（视部署方案而定）
- 用户输入需做长度与内容校验，避免异常请求
- 遵守微信小程序平台规范及 Coze API 使用条款
